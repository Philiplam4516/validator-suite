# A report is basically a list of uris with attached messages coming from different assertors. An obvious way to filter the list is to filter based on assertor (i.e. show me the list of uris that contain messages coming from that assertor and only display these messages). We could also filter based on number of errors (e.g. only urls with more than 50 errors) or other criterions, and compose filters. For the first version tab presentation based on assertors might be enough.
